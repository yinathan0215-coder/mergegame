# Title (로비) 화면 — 레이아웃 & 배경

> 과제 요구 ④-b — 에이전트가 추론하기 어려운 **시작 화면(로비) 구조**를 못 박는다. 씬 흐름(Title이
> 어디서 들어와 어디로 나가는지)은 [[../20-core-loop/screen-flow]], Pool In-Game HUD는 [[layout]].

세로형(9:16) 화면 전체에 **Galaxy background + 태양계 공전 배경**을 깔고, 그 위에 로비 UI를 얹는다.

## 1. 배경 — 태양계 공전

- 화면 전체를 채우는 **공유 Galaxy background**([[galaxy-background]]) 위에 태양계가 공전하는 애니메이션.
  정적 배경은 캐주얼 게임풍 우주 물결 배경을 담당하고, 별·점광·반짝임은 런타임 루핑 효과로 재생한다.
- **Pool In-Game과 동일한 행성 리소스**(`PlanetFactory`/행성 스프라이트)를 재사용한다 — 중앙에
  태양(9단계), 그 둘레를 **행성 8종(수성~목성, 1~8단계) 전부**가 서로 다른 반지름·속도의 궤도로
  공전한다. 궤도는 **원형에 가까운 낮은 이심률**(약한 원근 타원)로 그린다.
- **깊이(z) 정렬은 y 기준이며 태양도 포함된다**: 천체는 화면상 y가 클수록(아래/앞) 위에, 작을수록
  (위/뒤) 아래에 그려진다 — 행성이 궤도 앞쪽을 지날 땐 태양 위로, 뒤쪽을 지날 땐 태양 뒤로 가린다.
- 순수 **렌더 연출**이다(물리 시뮬레이션 아님). 톤은 [[art-direction]](우주·와인·골드), 행성
  색·패턴은 [[planet-art]]를 따른다. 행성 사다리 순서는 [[../10-concept/index]].

## 2. UI 배치 (앞면, 위→아래)

```
 [코인 + ]              ( 게임 머니 / 부가 슬롯 )            [⚙ 설정]   ← 상단 바
                          👑  최고 점수                                ← Play 위
 [일일 미션]            ┌──────────────┐            [출석 체크]
                       │   ▶  게임 시작  │                              ← 중앙
 [  상점  ]            └──────────────┘            [행운의 돌림판]
                        🪐  현재 점수(이어하기)                          ← Play 아래
                       (( Galaxy ) Fantasy )                          ← 하단 토글
```

### 2-1. 상단 바
| 위치 | 요소 | 동작 |
|---|---|---|
| **좌상단** | **게임 머니**(코인 수치 + `+`) | 표시용. 부가 재화/아바타 슬롯은 레퍼런스 기준 표시용 placeholder(기능 없음) |
| **우상단 끝** | **설정 버튼**(⚙ 기어) | 누르면 **설정 창을 연다**(프레스 피드백 포함). 설정 창 내부 항목은 이 프로토타입 범위 밖 — 차후 별도 스펙 |

### 2-2. 중앙 컬럼
| 순서 | 요소 | 내용 |
|---|---|---|
| Play **위** | **최고 점수**(👑 왕관 + 수치) | 역대 최고 점수 |
| 중앙 | **Play 버튼**(▶ 게임 시작) | 누르면 Pool In-Game으로 전환(`PLAY_CLICKED`, [[../20-core-loop/screen-flow]]). 진행 중 세션이 있으면 이어하기 |
| Play **아래** | **현재 점수**(🪐 현재 최고 도달 행성 아이콘 + 수치) | 진행 중(이어하기) Pool 세션의 현재 점수 |

**최고 점수·현재 점수는 모두 `localStorage`에 저장**되어 새로고침 후에도 유지된다(현재 점수 =
이어하기). 게임 오버가 없어 Pool 세션은 종료되지 않고 이어진다.

### 2-3. 좌·우 사이드 버튼
| 변 | 위(top) | 아래(bottom) |
|---|---|---|
| **왼쪽** | **일일 미션** | **상점** |
| **오른쪽** | **출석 체크** | **행운의 돌림판** |

사이드 버튼 4개는 모두 로비 셸 버튼이다 — 프로토타입에서는 패널 placeholder 진입(또는 무동작)이며
**핵심은 눌렀을 때의 버튼 피드백**(§3)이다.

### 2-4. 하단 토글 — Galaxy | Fantasy
- 화면 하단 중앙에 **알약(pill) 토글** 하나. 두 세그먼트 `Galaxy`·`Fantasy`를 담고, 활성 세그먼트
  위에 **하이라이트 캡슐**이 놓인다.
- **기본 활성 = Galaxy**(왼쪽 세그먼트에 하이라이트).
- 탭하면 하이라이트 캡슐이 반대 세그먼트로 **좌우 슬라이드 이동**한다(이동 애니메이션 필수).
- **기능은 시각 전환뿐**이다 — Fantasy로 옮겨도 연결된 모드/콘텐츠가 없어 게임 동작은 바뀌지
  않는다(테마 자리표시자, [[../20-core-loop/screen-flow]]).

## 3. 버튼 피드백 (공통 UX 규칙)

**모든 인터랙티브 버튼**(Play·사이드 4개·토글·설정·코인 `+`)은 **눌렀을 때 시각 피드백**을 준다.
연출 타이밍/스케일 수치의 SSoT는 `game/src/data/balance.json`(`juice.buttonPress`)이고, 규칙
정본은 [[feedback-effects]] §5다. 기본 규칙: pointerdown에 살짝 **눌리고(스케일 축소)**, release에
**탄성 복귀**(살짝 오버슈트 후 원래 크기)한다.

## 4. 이 화면에 두지 않는 요소

- **순위(랭킹) 위젯** — Title에서는 두지 않고, 그 자리를 **최고 점수**(👑)가 대신한다.
- **하단 4버튼**(광고 보상 · 돼지저금통 · 보물상자 상점 · 특별 할인) — 토글이 그 자리를 대신한다.
- **Arcade 버튼**, **광고 카드(Play Now!)** — 프로토타입 Title 범위 밖.
