# 행운의 돌림판

> 문서 범위 ③(시스템). Title 로비 `행운의 돌림판` 버튼으로 여는 창. 코인은 [[meta-economy]],
> 팝업 틀은 [[../50-art-ux/popup-system]], 수치 SSoT는 [[../40-balancing/meta-economy]].

## Summary

코인을 걸고 돌리는 8칸 룰렛. 1회 회전에 **100 코인**을 소모하고, **10·25·50·75·100·120·150·250**
중 하나의 코인을 균등 확률로 돌려준다. 회전 시작 → 정지 입력 → 3초 감속 정착 → **코인 쏟아짐
연출**의 인터랙션이다.

## Details

### 구성
- **8칸**, 각 칸 보상 코인 = [[../40-balancing/meta-economy]] `wheel.segments` =
  **10 · 25 · 50 · 75 · 100 · 120 · 150 · 250**.
- **1회 비용 = 100 코인**(`wheel.cost`). 잔액 < 100이면 회전 버튼 비활성.

### 회전 인터랙션
1. **회전 시작**: 회전 버튼을 누르면 코인 100을 차감하고 룰렛이 **빠르게 등속 회전**한다. 버튼은
   `정지`로 바뀐다.
2. **정지 입력**: `정지`를 누른 순간 결과 칸이 **결정**된다 — 8칸 **균등 확률 랜덤**으로 하나 선택.
3. **감속·정착**: 선택된 칸까지 **3초(`wheel.decelMs`)에 걸쳐 ease-out 감속**하며 그 칸의 포인터
   위치에 정확히 정지한다. 감속 구간엔 추가 난수가 없다(**결정론적** — 결과는 정지 순간 고정).
4. **지급**: 정지하면 해당 칸의 코인을 **쏟아짐 연출로 적립**한다(아래 §보상 연출).

### 보상 연출 (코인 와르르 → 포물선 비행)
- 룰렛이 멈추면 결과 코인은 **쏟아짐 연출을 통해 적립**된다 — 룰렛 중심에서 **코인 스프라이트가
  랜덤 좌표로 와르르 쏟아지고, 제각각 방향으로 중력 낙하**한다.
- 쏟아지는 스프라이트 수 = **결과값 ÷ 5**(`wheel.coinsPerPourSprite` = 5). 예: 결과 250 → 50개,
  10 → 2개.
- **낙하 1초 → 포물선 비행:** 약 **1초간 흩뿌려 낙하**한 뒤 **낙하를 멈추고**, 각 코인이 그 자리에서
  **포물선을 그리며 왼쪽 상단 코인 표시 위치로 날아간다**.
- **도착 시 적립(롤링):** 지급량(결과값)은 정지 즉시 잔액에 더해지되, **표시 숫자는 코인이 도착할 때
  비로소 1씩 굴러 오른다**(코인이 날아드는 동안 카운트업) — 최종 표시는 **이전 + 결과값**.
- **딤 위 코인 표시:** 돌림판 팝업이 열려 있는 동안 **왼쪽 상단 코인 표시를 딤보다 위(z-order 상승)**
  에 올려, 날아온 코인이 적립되는 과정이 딤에 가려지지 않고 보인다. 이는 Title·Pool In-Game 어느
  쪽에서 돌림판을 열든 동일하게 적용된다.

### 표시
- 팝업 틀([[../50-art-ux/popup-system]])을 **BG 패널 없이**(`hasBg=false`) 쓴다 — 딤 위에 제목
  "행운의 돌림판" + 룰렛 + 회전/정지 버튼만. X 버튼으로 닫는다.
- 포인터(고정 화살표)는 룰렛 상단, 결과 칸은 포인터 아래에 멈춘다.
- **회전/정지 버튼**(입체감 있는 공통 버튼, [[../50-art-ux/popup-system]] 버튼 규칙): 대기 시 왼쪽에
  **[코인 아이콘] 위 / [비용 100] 아래**로 세로 정렬한 비용 + 오른쪽에 **"회전"** 텍스트. 회전 중에는
  버튼 안에 **"정지"** 텍스트만 표시한다.
