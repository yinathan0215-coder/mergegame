# 레이아웃 & HUD — 9:16 세로 보드

> 과제 요구 ④-b — 에이전트가 추론하기 어려운 화면 구조/HUD/범위 판단을 못 박는다.
>
> **준수 기준(방법론):** [[../90-methodology/layered-rendering]] (레이어·가독성).
>
> 보드 내부 레이어 구조(배경색·아웃라인·플레이그라운드·발사대·플레이 영역·일방향 경계)는
> [[screen-structure]]가 정본. 이 페이지는 **화면 비율 + HUD 배치**를 정본으로 둔다.

## 1. 화면 비율

**결정: 모바일 세로 우선, 기준 화면비 9:16(세로). 배경/전경 2-레이어 fit(단일 캔버스).** 캔버스는
**뷰포트 크기**로 두고, 그 안을 두 스케일 그룹으로 나눈다 — 배경은 cover, 전경은 contain.

- **전경(인게임/UI)** — 보드·발사대·HUD·로비 버튼: **contain**(`min(vw/W, vh/H)`)으로 9:16 안에 전부
  중앙 정렬·**잘림 없이** 보인다. 화면을 채우려고 전경을 잘라 키우지 않는다.
- **배경(은하·태양계)**: **cover**(`max(vw/W, vh/H)`)로 **뷰포트 가장자리까지** 채운다(넘치는 부분은 잘림).
- 두 그룹을 **뷰포트 크기 단일 캔버스**에 둔다(배경 cover 그룹 + 전경 contain 그룹) → **캔버스는 1개**로
  유지한다. 배경이 화면 전체를 덮으므로 바깥 단색 여백 없이 가장자리까지 배경이 보인다([[screen-structure]]).

> 지시문의 "16:9"는 **세로형 9:16으로 해석**한다 — 첨부 레퍼런스 이미지가 세로형이고, 하단
> 발사대 + 위로 쌓이는 보드 구조가 세로를 요구한다. (가로형이 의도였다면 재확인 필요.)

## 2. HUD 배치 (상단 바)

상단에 3분할 HUD 바를 둔다. **친구(초대)·프리미엄 버튼은 제외**한다. Pool In-Game의 상단 **왼쪽
모서리에는 코인 잔액**을 Title 상단과 **동일한 위치·컴포넌트**로 표시하고, **오른쪽 모서리는 메뉴
버튼만** 둔다. 랭킹 표시는 두지 않는다.

| 위치 | 요소 | 비고 |
|---|---|---|
| **왼쪽 상단** | **뒤로가기 버튼**(`exit.png`) | 누르면 **Title 화면으로 복귀**한다(Pool In-Game→Title, [[../20-core-loop/screen-flow]] EXIT). 머니 표시는 두지 않는다 |
| **중앙 상단** | **Score**(현재 점수, 큰 수치), **최고 점수**(👑 왕관 + 작은 수치, Score 위) | **Score는 HUD 바 수평 중앙 정렬·상단 배치**, 최고 점수(👑+수치)는 Score 바로 위 중앙. 현재 Score가 강조(대형) |
| **오른쪽 상단** | **메뉴 버튼**(`menu.png`) | 누르면 **인게임 드롭다운 메뉴**(아이콘 4개)를 토글한다(아래 §2-c). 랭킹 표시는 두지 않는다 |

상단 좌·우 버튼은 [[button-system]]을 따른다. 나가기와 햄버거 메뉴는 [[title-icons]]의 PNG **아이콘만** 상단 바 위에 띄운다(코너 버튼은 아이콘 자체가 탭 영역 — 배경 박스를 두지 않는 점이 Title 사이드 버튼과 다르다).

발사대 안에 **현재 발사 행성 1개**를 보여준다([[../30-systems/launch-queue]]).

## 2-c. 메뉴 버튼(≡) → 인게임 드롭다운 (아이콘 단축 메뉴)

Pool In-Game 상단 오른쪽 **메뉴 버튼(≡) 바로 아래**로 **아이콘만 있는 세로 버튼 리스트**가 펼쳐진다
(드롭다운). Title 로비의 메타 버튼과 **같은 기능을 인게임에서 바로 호출**하는 단축 메뉴다.

- **위치:** 리스트는 **≡ 버튼 바로 아래**(버튼 중심에 세로로 정렬)에 매달려 펼쳐진다.
- **항목(4개, 아이콘만):** **일일 미션 · 출석 체크 · 행운의 돌림판 · 설정** — Title 로비 버튼과 동일한
  아이콘([[title-icons]])을 쓰며 라벨 텍스트는 없다. **4개 아이콘은 하나의 공통 딤드 박스**(검은 반투명
  라운드)로 묶는다 — 항목별 박스가 아니라 리스트 전체가 한 박스다.
- **동작:** 각 항목은 **Title의 같은 버튼과 동일한 기능**을 한다 — 해당 메타 팝업([[popup-system]])을 연다
  (일일 미션·출석·돌림판). **설정**은 Title 설정 버튼과 **동일한 동작을 공유**한다(설정 팝업을 붙이면
  Title·인게임 양쪽에 함께 연결된다).
- **열기/닫기:** 메뉴 버튼을 **다시 누르거나** 리스트 **바깥(다른 화면)을 누르면** 닫힌다. 한 항목을
  누르면 그 팝업이 열리며 리스트는 닫힌다(한 번에 하나).
- **레이어:** 드롭다운은 HUD(전경 contain)에 속한다. 열려 있는 동안 **바깥 탭을 받는 투명 영역**이
  보드 입력(발사·조준)을 가로채 닫기 전용으로 쓴다(오조작 방지). 항목·메뉴 버튼만 그 위에서 눌린다.

## 2-b. 하단·측면 HUD (모드별)

상단 바 외에 보드 위 빈 영역에 게임 모드([[../20-core-loop/game-modes]]) 정보를 얹는다.

위젯은 보드 하단 스트립에 얹는다 — **위젯의 수직 중앙을 보드 아웃라인 끝(발사대 하단)에 맞춰**
플레이 영역을 가리지 않게 한다.

| 위치 | Infinite | Stage |
|---|---|---|
| **하단 좌측** | 남은 **Count** + 큐의 **Next** 미리보기 (**가로로 나란히**) | 남은 **Count** + 큐의 **Next** (가로로 나란히) |
| **하단 우측** | **행성 충전 버튼**(회전 지구 + `Planet Charge`, [[../30-systems/planet-charge]]) | **목표 행성**(이름 + 회전하는 행성 이미지, [[../30-systems/stage-mode]]) |

- **Count/Next**: 남은 발사 횟수([[../30-systems/launch-count]])와 다음 발사 행성 미리보기
  ([[../30-systems/launch-queue]])를 **가로로 나란히** 두고, 하단 우측 버튼과 **수직 중앙을 맞춘다**.
- 하단 우측 위젯은 모드 전용이며 같은 자리를 공유한다(Infinite=충전 버튼, Stage=목표 행성).
- 모든 하단 위젯은 보드/발사대보다 **아래**에 배치해 플레이 영역을 가리지 않는다.

## 3. 화면 영역 구조 (위→아래)

| 영역 | 위치 | 표시 요소 |
|---|---|---|
| HUD 바 | 최상단 | 좌: 뒤로가기(→Title) / 중앙: 최고점수·Score / 우: 메뉴 |
| 보드(playground) | HUD 아래 대부분 | 아웃라인 + 플레이그라운드 배경 + 행성 + **방패형 보드/발사대**([[screen-structure]]) |
| 하단 스트립(좌) | 보드/발사대 아래 | 남은 Count + Next 미리보기(가로 나란히, 모드 공통) |
| 하단 스트립(우) | 보드/발사대 아래 우측 | 충전 버튼(Infinite) / 목표 행성(Stage) |
| 발사대 | 보드 **둥근 볼록 끝** | 현재 발사 행성 · 드래그 조준선 · 원형 강조 + 장식 아크 |
