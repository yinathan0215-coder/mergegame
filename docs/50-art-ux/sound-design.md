# 사운드 디자인 — 컨셉 & 효과음/UI 사운드 가이드

> 과제 요구 ④-b(에이전트가 추론하기 어려운 영역). **아트 방향([[art-direction]])의 청각 짝**:
> 어떤 사건에 어떤 소리가 나는지, 톤은 어떤지, **동시에 너무 많은 소리가 겹치지 않게** 하는 UX
> 규칙을 못 박는다. 구현(사운드 매니저·합성)은 [[../60-implementation/sound-manager]], 수치 SSoT는
> `game/src/data/balance.json`(`sound`)이다.

## 1. 컨셉 / 톤 (결정)

- **캐주얼 모바일 게임풍**: 짧고 **가볍고 만족스러운** SFX. 거슬리거나 길게 끌지 않는다(아트의
  캐주얼 2.5D 톤과 동일 성격).
- **효과음(SFX) + UI 사운드** 범위. **BGM(배경음악)은 이번 범위 밖**(차후 옵션, `draft`).
- **만족감의 핵심은 머지**: 합성 시 **등급이 올라갈수록 피치가 높아지는** 팝으로 성장 보상을 청각화한다.
- **생성 방식 = 절차적 합성(Web Audio)**: 프로토타입은 오실레이터+엔벨로프(+노이즈)로 **코드에서
  소리를 생성**한다 — 에셋 파일·라이선스·다운로드 없이 오프라인 동작. 차후 실제 음원 에셋으로 교체
  가능(매니저 API는 동일).

## 2. 사운드 카탈로그 (사건 → 소리)

| id | 발생 사건 | 느낌 | 우선순위 |
|---|---|---|---|
| `uiPress` | 버튼 누름(로비·HUD·모달 OK·토글) | 부드러운 클릭/팝 | 중 |
| `toggle` | Galaxy/Fantasy 토글 전환 | 짧은 틱 | 중 |
| `play` | 게임 시작(Play) | 밝은 확정 휘릭 | 높음 |
| `popupOpen` | 팝업/모달 등장 | 부드러운 차임 | 높음 |
| `popupClose` | 팝업 닫기(X) | 짧은 틱 | 중 |
| `launch` | 발사(release) | 휘릭(whoosh), 파워↑→피치↑ | 중 |
| `wall` | 벽(inner line)·발사대 충돌 | 가벼운 틱 | **낮음** |
| `ballHit` | 행성–행성 충돌(머지 아님) | 짧은 탁(clack) | **낮음** |
| `merge` | 합성 | 만족스러운 팝, **생성 등급↑→피치↑** | 높음 |
| `comboMilestone` | 콤보 5의 배수 도달 | 상승 아르페지오(반짝) | 높음 |
| `unlock` | 새 등급 해금 모달 | 밝은 차임/팡파레 | 높음 |

## 3. UX 규칙 (못 박음)

### 동시 재생 제한 — "너무 많은 소리가 겹치지 않게"
- **동시 보이스 상한 `maxVoices`**: 활성 보이스가 상한이면 **새 재생은 우선순위로 판정** — 더 낮은
  우선순위 보이스를 끊고 들어가거나, 낮은 우선순위 새 소리는 **생략**한다.
- **사운드별 스로틀 `throttleMs`**: 같은 `id`는 최소 간격 안에 재트리거하지 않는다. **다발로 터지는
  소리(`wall`·`ballHit`)는 짧은 스로틀로 솎아내** 충돌 폭주 시에도 한두 번만 들리게 한다(가장 중요한 절감).
- **우선순위**: `merge`·`unlock`·`comboMilestone`·`play`·`popupOpen` = 높음(보존), `wall`·`ballHit` = 낮음
  (먼저 끊김/생략). 강한 충돌로 벽·볼 소리가 폭주해도 머지·해금 소리는 항상 들린다.

### 충돌음 가청도 — "낮은 우선순위 ≠ 안 들림"
- `wall`·`ballHit`는 **폭주 시 솎이는** 대상(낮은 우선순위·짧은 스로틀)이지만, **한 번 들릴 때는
  발사·머지에 묻히지 않게 충분히 큰 레벨**로 들려 충돌 타격감을 준다. 충돌음을 절감 대상으로만 보고
  게인을 너무 낮추면 "충돌음이 빠진" 것처럼 들린다 — `wall`·`ballHit` 게인은 `launch`·`merge`와 **동등
  수준**(balance.json `sound`의 wall/ballHit `gain` 참조)을 유지한다.

### 음소거 / 자동재생
- **마스터 음소거** 지원 — 플레이어 진입점은 [[../30-systems/settings]] 설정 팝업의 **사운드** 버튼
  (= 뮤트 토글). **기본 ON**(소리 켜짐). 음소거 상태는 `localStorage`(`ppm.muted`)에 영속. (볼륨 단계
  조절은 이번 범위 밖 — 뮤트만.)
- 브라우저 자동재생 정책: `AudioContext`는 **첫 사용자 입력(pointerdown)에서 resume**한다(그 전엔 무음).
- 음소거 중에도 게임 로직·연출은 정상 동작한다(사운드는 렌더처럼 **부수 효과**, 시뮬레이션을 되먹이지 않음).
