# 입력 & UX — press-drag-release 조준 (§8.2)

> 과제 요구 ④-b — UX 판단(조준선·플레이 영역·직관성)을 못 박는다.
>
> **준수 기준(방법론):** [[../90-methodology/layered-rendering]] (입력 우선순위·가독성).

## 1. 입력 & UX (§8.2)

**결정: 마우스 + 터치 모두 지원.** 버튼 없이 발사대 행성을 직접 조작한다.

발사 인터랙션 흐름:

1. **발사대 행성을 누른 상태(press)에서 드래그**하면 조준 상태로 진입한다.
2. 조준 중에는 **반투명 조준선이 첫 충돌 지점까지** 보인다(아래 §2 충돌 궤적 예측).
3. 발사 방향은 드래그 방향의 **반대 방향**이다(당겨서 쏘는 새총/풀 큐 감각).
4. **파워는 힘 게이지로** 표시한다 — 드래그가 길수록 게이지가 찬다(거리→파워 정규화·최대속도는
   [[../40-balancing/index]]). 조준선 길이는 파워가 아니라 충돌까지의 거리를 나타낸다.
5. **손을 놓으면(release) 발사**된다.
6. **조준 중 취소 기능은 프로토타입 필수 범위가 아니다**(만들지 않아도 됨).

> 발사대 고정 위치·쿨다운·큐 갱신 규칙은 [[../20-core-loop/index]], 거리/파워/쿨다운 숫자는
> [[../40-balancing/index]].

## 2. 조준선 = 충돌 궤적 예측

조준선은 **현재 발사 방향으로 행성이 날아가 처음 부딪히는 지점까지** 그린다 — 부딪히는 대상은
**충돌 경계 벽(inner line)과 다른 행성** 모두다. 예측은 **실제 충돌과 정확히 일치**한다: 발사
행성 반지름만큼 벽은 안쪽으로·다른 행성은 바깥으로 부풀려, 디스크가 실제로 닿는 지점에서 멈춘다.

- **첫 충돌 지점**에 마커를 찍는다.
- 그 지점의 **반사선(튕기는 각)**을 이어 그리되, **다음 충돌(벽·행성)에서 끊는다**.
- 반사선의 **끝은 점점 흐려진다**(페이드 아웃).

조준선 길이는 드래그 양이 아니라 **충돌까지의 실제 거리**이며, 파워는 힘 게이지로 본다. 충돌 형태는
[[../30-systems/collision-shape]](원형 디스크), 메카닉 정본은 [[../30-systems/launcher]].

## 3. UX 판단 (§13)

1. **조준선은 실제 발사 방향과 정확히 일치해야 한다.**
2. **조준선은 첫 충돌 지점에서 끝나므로** 보드 경계 안에 자연히 머문다.
3. **UI는 플레이 영역(보드)을 가리지 않는다.**
4. **발사대 행성은 작아도 패턴이 읽혀야 한다**(작은 크기에서도 색+패턴 식별 가능하게 단순화).
5. **모바일 세로 화면에서 버튼이 없어도 발사 방법이 직관적**이어야 한다(press-drag-release만으로).

> 물리 손맛 휴리스틱(마찰·반발·최소속도·초반 5발 안 1합성)은 아트/UX가 아니라
> [[../40-balancing/index]]의 영역이다.
