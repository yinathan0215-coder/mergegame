# Data-driven Design: 밸런스 수치를 코드가 아니라 표로 고정

> 적용: [[../40-balancing/index|40-balancing]]. 출처: [[index|방법론 인덱스]].

## 목적
밸런스·콘텐츠·진행 조건·UI 텍스트를 코드 로직에서 분리한다. 수치가 하드코딩되면 검증·수정이
느려지고, 에이전트가 임의 값을 만들거나 중복 작성한다. **조정 가능한 값은 단일 데이터 출처에**.

## 데이터로 분리해야 할 항목
| 범주 | 예시 |
|---|---|
| 세션 설정 | 세션 길이, 시작 재화, 초기 체력, 목표 점수 |
| 경제 | 생산량, 보상량, 비용, 증가율, 재화 이름 |
| 병합/성장 | 레벨별 가치, 다음 레벨 조건, 해금 조건 |
| 스폰 | 등장 간격·수, 난이도 증가율, 위치 규칙 |
| 전투/압박 | 적 체력, 피해량, 이동 속도, 공격 간격 |
| UI | 버튼·안내·결과 문구 |
| 튜토리얼 | 단계별 안내, 완료 조건, 강조 대상 |
| KPI | 첫 보상/성장 목표 시간, 루프 반복 목표 |

## 밸런스 테이블 작성 기준 — 값에 **의도**를 함께
| 항목 | 값 | 단위 | 의도 |
|---|---:|---|---|
| 시작 코인 | 20 | coin | 첫 행동 즉시 가능 |
| 기본 생산량 | 1 | coin/sec | 방치 보상을 10초 안에 인지 |
| 첫 업그레이드 비용 | 10 | coin | 30초 이내 첫 성장 유도 |
| 세션 길이 | 180 | sec | 3분 안에 핵심 루프 검증 |
| 첫 압박 발생 | 45 | sec | 루프 이해 후 위험 도입 |
| 첫 실패 가능 | 90 | sec | 성장 선택 필요성 인지 |

> 위 수치는 **방법론의 예시값**이다. merge 게임 실제 값은 [[../40-balancing/index|40-balancing]]에서 확정.

## 데이터 구조 기준 (이 단위로 분리)
| 파일/섹션 | 포함 |
|---|---|
| `gameConfig` | 화면 크기, 세션 길이, 초기 상태, 디버그 옵션 |
| `balanceConfig` | 비용, 보상, 생산량, 성장 곡선 |
| `spawnConfig` | 스폰 간격, 등장 조건, 난이도 증가 |
| `entityConfig` | Entity 타입별 기본 Component 값 |
| `uiTextConfig` | 버튼, 튜토리얼, 결과 문구 |
| `kpiConfig` | 검증 기준·목표 수치 |

## 공식 작성 기준 (계산식이 필요할 때)
| 항목 | 공식 예시 | 설명 |
|---|---|---|
| 업그레이드 비용 | `baseCost * growthRate^(level-1)` | 레벨↑ 비용↑ |
| 생산량 | `baseIncome * incomeMultiplier^level` | 성장 체감 |
| 적 체력 | `baseHp + hpPerWave * wave` | 시간 압박 |
| 보상량 | `baseReward + levelBonus * level` | 성장 보상 |

공식엔 기준값·증가율·반올림 규칙을 함께 정의한다.

## 안티패턴
- 같은 비용이 코드·UI·튜토리얼에 각각 다르게 하드코딩
- 수치에 단위가 없음
- `speed = 5` 처럼 의도 없는 숫자만
- 레벨 공식이 코드에만 있고 문서엔 없음
- 에이전트가 적당한 값을 임의로 정하게 둠

## 완료 기준
- 모든 조정 가능한 수치가 한 곳에서 확인됨
- 각 수치에 단위·의도가 적힘
- UI 표시 수치와 로직 수치가 같은 출처 참조
- 첫 보상/성장/압박/실패 시점이 수치로 검증 가능
- 밸런스 조정에 핵심 로직 코드 수정 불필요
