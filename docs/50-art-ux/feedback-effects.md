# 피드백 연출 (Juice)

> 머지·점수의 **체감 피드백**을 못 박는다(에이전트가 임의로 결정하면 안 되는 연출). 타이밍/크기
> 수치 SSoT는 `game/src/data/balance.json`(`juice`). 렌더 전용 — 시뮬레이션을 되먹이지 않는다
> ([[../90-methodology/layered-rendering]]). 추가 레이어: `effectLayer`(행성 위, 조준선 아래) +
> 점수 연출은 `uiLayer`(HUD).

## 1. 머지 스케일 팝 (생성 연출)
머지로 새 행성이 생성될 때, 스프라이트가 **작게 시작 → 크게 부풀었다 → 원래 크기로** 정착하는
튀어오르는 팝. (`juice.mergePop`)

| 파라미터 | 값 | 의미 |
|---|---|---|
| `ms` | 260 | 팝 지속 |
| `startScale` | 0.35 | 시작 크기 |
| `peakScale` | 1.18 | 최대(오버슈트) 크기 |

- 곡선: 0~40% 구간 `startScale→peakScale`, 40~100% 구간 `peakScale→1.0`.
- **머지 결과 행성에만** 적용(발사·초기 랙 행성은 팝 없음).

## 2. 머지 발산 버스트 (강조 이펙트)
머지 위치에서 **반투명 원이 바깥으로 발산하며 페이드아웃**하는 링. (`juice.burst`)

| 파라미터 | 값 | 의미 |
|---|---|---|
| `ms` | 380 | 버스트 지속 |
| `scale` | 2.6 | 최종 반지름 = 결과 행성 r × 2.6 |
| `alpha` | 0.5 | 시작 불투명도(→0으로 페이드) |

- 색은 **생성된 행성의 메인 색**을 사용. 선 두께는 진행에 따라 가늘어진다.

## 3. 점수 오도미터 (1단위 스크롤)
Score 숫자는 목표값으로 **즉시 점프하지 않고 정수 단위로 빠르게 스크롤하며 상승**한다.
(`juice.scoreRoll`)

| 파라미터 | 값 | 의미 |
|---|---|---|
| `lerp` | 0.18 | 매 프레임 `shown += max(1, ceil(diff×lerp))` — 큰 차는 빠르게, 작은 차는 1씩 |

- 항상 정수로 증가, 목표 초과 없이 수렴. 최고점수(👑)는 목표값 기준 즉시 갱신.

## 4. 머지 +N 플로팅 점수
머지 시 획득 점수가 **`+N`** 텍스트로 **머지가 생성된 위치(보드 좌표)** 에 떠올라 위로 올라가며
천천히 사라진다. (`juice.scorePopup`)

| 파라미터 | 값 | 의미 |
|---|---|---|
| `ms` | 950 | 표시 시간(페이드까지) |
| `rise` | 46 | 떠오르는 높이(px) |
| `color` | `#ffe28a` | 텍스트 색 |
| `fontSize` | 22 | 텍스트 크기 |

- 충돌 점수는 플로팅을 만들지 않는다(오도미터로만 반영) — 머지만 강조.

## 5. 버튼 프레스 피드백
**모든 인터랙티브 버튼**(Title 로비의 Play·사이드·토글·설정, HUD 버튼 등)은 눌렀을 때 시각
피드백을 준다. pointerdown에 **눌리고**(스케일 축소), release에 **탄성 복귀**(오버슈트 후 정착)한다.
(`juice.buttonPress`)

| 파라미터 | 값 | 의미 |
|---|---|---|
| `downScale` | 0.92 | pointerdown 시 축소 크기 |
| `upScale` | 1.04 | release 직후 오버슈트 크기 |
| `ms` | 140 | 복귀(오버슈트→1.0) 지속 |

- press 상태는 즉시 `downScale`로, release는 `upScale`로 튄 뒤 `ms`에 걸쳐 1.0으로 정착.
- 적용 대상은 버튼(UI)뿐 — 행성·보드에는 적용하지 않는다([[title-screen]] §3).
- **공통 모듈:** 단일 헬퍼(`attachButtonFeedback`)로 모듈화되어 Title 로비·HUD·**해금 모달 OK 버튼**
  ([[../30-systems/tier-unlock]]) 등 **모든 탭 가능한 컨트롤**에 동일한 느낌으로 적용된다. 복귀
  애니메이션은 게임 루프가 멈춰 있어도(모달 일시정지) 돌아가야 하므로 렌더 루프와 독립적으로 구동된다.

## 6. 충돌 히트 이펙트 (방향성 스프레이)

점수에 충족한 충돌(법선 impact ≥ `scoring.minImpact`) 지점에서 **튕기는(반사) 방향으로 짧은
스파크가 발산**한다. 머지 발산 버스트(§2, 방사형 링)와는 **다른** 방향성 이펙트다. (`juice.hitBurst`)

| 파라미터 | 값 | 의미 |
|---|---|---|
| `ms` | 260 | 지속(빠름) |
| `len` | 26 | 스파크가 뻗는 길이(px) |
| `streaks` | 5 | 발산 줄기 수 |
| `spreadDeg` | 34 | 튕기는 방향 기준 ± 퍼짐각 |
| `alpha` | 0.7 | 시작 불투명도(→0 페이드) |

- 방향 = 충돌 후 **튕기는(반사) 방향**(충돌 법선에 대한 입사 속도 반사). 색은 흰색(조준선 색).
  발생 조건·점수 규칙은 [[../30-systems/scoring-combo]] · [[../40-balancing/combo-scoring]].

## 7. 성능 (이펙트 최적화)

충돌 히트 이펙트는 자주 발생하므로 프레임 드랍을 막는 처리를 둔다.

- **최소 충돌 힘 게이트:** `impact < scoring.minImpact` 접촉은 점수·이펙트 모두 생략한다 — 약한
  접촉 폭주를 원천 차단(가장 큰 절감).
- **동시 이펙트 상한:** 동시 버스트·플로팅 수에 상한을 둬 초과분은 생략한다(완성도보다 성능 우선).
- **Graphics 풀링:** 버스트용 `Graphics`를 파괴하지 않고 재사용(풀)해 매 충돌마다의 할당/GC를 없앤다.
- **정적 지오메트리 캐시:** 조준 궤적이 쓰는 inner-line 폴리라인은 한 번 계산해 캐시한다.

## 8. 콤보 카운터 (머지 체인)

연속 머지를 세는 **시각 카운터**다. **점수 배율이 아니다** — 점수 규칙은 그대로(충돌 +N·머지 기본 점수,
배율 콤보는 제거됨 [[../40-balancing/decisions/2026-06-28-remove-combo]]). (`juice.combo`)

- **유지 타이머 = 절대 시간 `holdMs`(5초).** 머지가 일어날 때마다 **타이머를 0으로 리셋**한다.
- **카운트:** 타이머가 살아 있는 동안 새 머지가 성공하면 **콤보 횟수 +1**, 타이머가 지나면 체인이 끊겨
  다음 머지부터 다시 **1**로 시작한다(첫 머지 = 1).
- **표시:** **행성 뒤(배경 위)** 레이어에 `Combo` 라벨 + 숫자를 **중앙 정렬**로 그린다. 숫자는 점수와 같이
  **1단위로 상승**한다(오도미터, §3과 동일).
- **사라짐:** 마지막 머지 후 **`fadeStartMs`(3초)까지는 풀 불투명**, **3~5초 구간에서 선형으로 투명해져**
  `holdMs`(5초)에 완전히 사라진다(체인 종료).

### 콤보 마일스톤 보너스
콤보가 **`step`(5)의 배수에 도달**할 때마다(5·10·15·20·25…) **큰 보너스 점수**(`콤보값 × bonusPer`)를
획득한다([[../30-systems/scoring-combo]]). **콤보는 어렵게 쌓이므로 보너스는 상당히 크다.**

- 획득 시 **화면 중앙**에 **2줄 플로팅**을 띄운다 — 위에 **`combo M`**(작은 글씨 `bonusLabelSize`), 아래에
  **`+N`**(큰 글씨 `bonusFontSize`). 머지 +N(§4)처럼 떠오르며 사라지되 **유지시간이 더 길다**(`bonusMs`).
  색은 `bonusColor`.
- 보너스는 일반 점수처럼 Score(오도미터)에도 합산된다.

| 파라미터 | 값 | 의미 |
|---|---|---|
| `holdMs` | 5000 | 콤보 유지 타이머(머지마다 리셋, 절대 시간) |
| `fadeStartMs` | 3000 | 이 시점 이후부터 페이드아웃 시작 |
| `step` | 5 | 보너스 마일스톤 간격(5의 배수) |
| `bonusPer` | 400 | 마일스톤 보너스 = 콤보값 × bonusPer |
| `bonusFontSize` | 40 | 중앙 보너스 `+N`(아래 줄) 폰트 |
| `bonusLabelSize` | 22 | `combo M`(위 줄) 폰트(더 작게) |
| `bonusMs` | 1800 | 중앙 보너스 유지시간(머지 950보다 김) |
| `bonusRise` | 56 | 보너스 떠오르는 높이(px) |
| `bonusColor` | `#ffd23f` | 보너스 텍스트 색 |
| `labelSize` | 24 | "Combo" 라벨 크기 |
| `numberSize` | 72 | 숫자 크기 |
| `color` | `#ffffff` | 카운터 텍스트 색 |
| `alpha` | 0.22 | 카운터 불투명도(배경 워터마크) |
| `cyFrac` | 0.46 | 플레이 영역 세로 위치(0~1) |
| `gap` | 6 | 라벨–숫자 세로 간격(px) |
