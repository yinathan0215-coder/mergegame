# Title (로비) 화면 — 레이아웃 & 배경

> 과제 요구 ④-b — 에이전트가 추론하기 어려운 **시작 화면(로비) 구조**를 못 박는다. 씬 흐름(Title이
> 어디서 들어와 어디로 나가는지)은 [[../20-core-loop/screen-flow]], Pool In-Game HUD는 [[layout]].

세로형(9:16) 화면 전체에 **Galaxy background + 태양계 공전 배경**을 깔고, 그 위에 로비 UI를 얹는다.

## 1. 배경 — 태양계 공전

- 화면 전체를 채우는 **공유 Galaxy background**([[galaxy-background]]) 위에 태양계가 공전하는 애니메이션.
  정적 배경은 캐주얼 게임풍 우주 물결 배경을 담당하고, 별·점광·반짝임은 런타임 루핑 효과로 재생한다.
  **Title 한정**으로 이 **은하수 배경(GalaxyBackground = 우주 이미지 + 반짝임 효과)만** cover로 뷰포트
  상하 가장자리까지 확장해 여백을 채운다. **태양계(공전)·로비 UI는 contain(9:16)** 으로 그 위에 얹혀
  잘리지 않으며, 캔버스 바깥/HTML 페이지 기본 배경은 평소대로 **단색(`outerBg`)** 이다([[layout]] §1).
- **Pool In-Game과 동일한 행성 리소스**(`PlanetFactory`/행성 스프라이트)를 재사용한다 — 중앙에
  태양(10단계), 그 둘레를 **태양계 행성 8종을 실제 거리 순서**(안쪽→바깥: 수성·금성·지구·화성·목성·
  토성·천왕성·해왕성)로 공전시킨다. 소행성(1단계)과 블랙홀(11단계)은 로비 태양계 공전 배경에는 넣지 않는다. 각 행성은
  공전과 동시에 **자전(자체 축 회전)**한다(행성마다 다른 속도·방향). **태양도 제자리에서 느리게 자전한다.**
- 궤도는 **살짝 눕힌(낮은 시점) 타원**이며, 태양계를 **모바일 세로 화면의 상하를 꽉 채우도록 크게
  확대**한다 — 바깥 궤도·행성은 화면 밖으로 넘어가도 된다(전체가 다 보일 필요 없음). 궤도 크기는
  화면 높이에 비례시켜 어떤 화면에서도 상하 여백 없이 채운다. **태양(중점)은 게임 시작 버튼보다 위에
  보이도록 궤도 중심을 화면 상단부에 둔다.**
- **깊이(z) 정렬은 y 기준이며 태양도 포함된다**: 천체는 화면상 y가 클수록(아래/앞) 위에, 작을수록
  (위/뒤) 아래에 그려진다 — 행성이 궤도 앞쪽을 지날 땐 태양 위로, 뒤쪽을 지날 땐 태양 뒤로 가린다.
- 순수 **렌더 연출**이다(물리 시뮬레이션 아님). 톤은 [[art-direction]](우주·네이비·골드), 행성
  색·패턴은 [[planet-art]]를 따른다. 행성 사다리 순서는 [[../10-concept/index]].

## 2. UI 배치 (앞면, 위→아래)

```
 [코인 + ]              ( 게임 머니 / 부가 슬롯 )            [⚙ 설정]   ← 상단 바
                          👑  최고 점수                                ← Play 위
 [일일 미션]            ┌──────────────┐            [출석 체크]
                       │  ▶ Game Start │                              ← 중앙(모드별 라벨)
 [  상점  ]            └──────────────┘            [행운의 돌림판]
                        🪐  현재 점수(이어하기)                          ← Play 아래
                       (( Infinite ) Stage )                          ← 하단 모드 토글
```

### 2-1. 상단 바
| 위치 | 요소 | 동작 |
|---|---|---|
| **좌상단** | **게임 머니**(코인 수치 + `+`) | 표시용. 부가 재화/아바타 슬롯은 레퍼런스 기준 표시용 placeholder(기능 없음) |
| **우상단 끝** | **설정 버튼**(`settings.png` 기어 리소스) | 누르면 **설정 창을 연다**(공통 팝업, 프레스 피드백 포함). 창 내부는 [[../30-systems/settings]] — **사운드(뮤트 토글)만 동작**, 나머지는 구색용 placeholder |

### 2-2. 중앙 컬럼
| 순서 | 요소 | 내용 |
|---|---|---|
| Play **위** | **최고 점수**(👑 왕관 + 수치) / **Stage 정보** | Infinite: 역대 최고 점수. **Stage 모드: 이 영역이 `Stage N`(스테이지 정보)로 대체된다** |
| 중앙 | **Play 버튼**(▶ `Game Start`) | 라벨은 **두 모드 공통 `Game Start`**. 누르면 **선택 모드**로 Pool In-Game 전환(`PLAY_CLICKED`, [[../20-core-loop/screen-flow]] · [[../20-core-loop/game-modes]]). 진행 중 세션이 있으면 이어하기 |
| Play **아래** | **현재 점수**(🪐 최대 머지 행성 아이콘 + 수치) | Infinite에서만 표시. **Stage 모드에서는 최고·현재 점수 UI를 숨긴다** |

**최고 점수 + 게임 시작 버튼은 검은색 반투명 사각 박스로 함께 묶어** 표시한다(현재 점수는 박스 아래·바깥).

Play 버튼은 [[title-icons]]의 버튼 몸체 이미지를 [[button-system]]의 9-slice 방식으로 사용한다. Infinite 모드에서는 `play-button.png`, Stage 모드에서는 색상만 짙은 보라색으로 다른 `play-button-stage.png`로 교체한다. 소스 이미지와 실제 로비 버튼 영역은 224x100 기준을 유지하고, 흰색 플레이 삼각형과 **공통 라벨 텍스트 `Game Start`**(두 모드 동일, [[../20-core-loop/game-modes]])만 버튼 안전영역 안에 런타임으로 얹는다. 라벨 텍스트는 하단 금색 베벨이 아니라 현재 버튼 면(파랑 또는 보라) 내부에 들어와야 하며, 위치 보정은 오버레이 기준점으로 해결한다. 눌림 전용 이미지는 두지 않고 공통 버튼 피드백으로 눌림 상태를 표현하며, 실사 재질이나 사진풍 조명은 쓰지 않는다.

**최고 점수·현재 점수는 모두 `localStorage`에 저장**되어 새로고침 후에도 유지된다(현재 점수 =
이어하기). 세션 종료는 게임 모드가 정하며([[../20-core-loop/game-modes]]), 진행 중 세션은
나가기(←)로 떠나도 보존돼 이어하기로 다시 보인다.

### 2-3. 좌·우 사이드 버튼
| 변 | 위(top) | 아래(bottom) |
|---|---|---|
| **왼쪽** | **일일 미션** | **상점** |
| **오른쪽** | **출석 체크** | **행운의 돌림판** |

사이드 버튼 4개는 모두 **아이콘 타일 + 라벨 카드** 형태(레퍼런스 이미지)이며 로비 셸 버튼이다 —
프로토타입에서는 패널 placeholder 진입(또는 무동작)이고 **핵심은 눌렀을 때의 버튼 피드백**(§3)이다.

아이콘은 [[title-icons]]의 PNG 리소스를 사용한다. 일일 미션=`daily-mission.png`, 상점=`shop.png`, 출석 체크=`check-in.png`, 행운의 돌림판=`lucky-wheel.png`. 각 이미지가 들어간 버튼 외형은 [[button-system]]을 따른다. **검은색 반투명 라운드 사각형은 아이콘 뒤 배경으로만 쓰고, 라벨 텍스트는 버튼 이미지 밑에 분리해서 둔다.** 코드에서 이모지, 폰트 글리프, 절차 그래픽으로 대체하지 않는다.

### 2-4. 하단 모드 토글 — Infinite | Stage
- 화면 하단 중앙에 **알약(pill) 토글** 하나. 두 세그먼트 `Infinite`·`Stage`를 담고, 활성 세그먼트
  위에 **하이라이트 캡슐**이 놓인다.
- 토글 중앙 위에는 작은 `Mode` 라벨을 둔다.
- **기본 활성 = Infinite**(왼쪽 세그먼트에 하이라이트).
- 탭하면 하이라이트 캡슐이 반대 세그먼트로 **좌우 슬라이드 이동**한다(이동 애니메이션 필수). 토글의 pill 구조와 형태는 바꾸지 않고, 하이라이트 색상만 Infinite=`#1f8efa`, Stage=`#4e1da9`로 바꾼다.
- 토글은 진입할 **게임 모드**를 고른다([[../20-core-loop/game-modes]]). **Stage로 옮기면** Play
  버튼 라벨은 `Game Start` 그대로 두고, **최고·현재 점수 UI가 사라지며 그 자리(최고 점수 영역)에
  `Stage N` 정보가 표시된다**(§2-2).

## 3. 버튼 피드백 (공통 UX 규칙)

**모든 인터랙티브 버튼**(Play·사이드 4개·토글·설정·코인 `+`)은 **눌렀을 때 시각 피드백**을 준다.
연출 타이밍/스케일 수치의 SSoT는 `game/src/data/balance.json`(`juice.buttonPress`)이고, 규칙
정본은 [[feedback-effects]] §5다. 기본 규칙: pointerdown에 살짝 **눌리고(스케일 축소)**, release에
**탄성 복귀**(살짝 오버슈트 후 원래 크기)한다.

## 4. 이 화면에 두지 않는 요소

- **순위(랭킹) 위젯** — Title에서는 두지 않고, 그 자리를 **최고 점수**(👑)가 대신한다.
- **하단 4버튼**(광고 보상 · 돼지저금통 · 보물상자 상점 · 특별 할인) — 토글이 그 자리를 대신한다.
- **Arcade 버튼**, **광고 카드(Play Now!)** — 프로토타입 Title 범위 밖.
