# Planet Pool Merge 프로토타입 설계

## 1. 목적

HTML5 기반으로 플레이 가능한 세로형 물리 머지 프로토타입을 만든다. 플레이어는 하단 고정 발사대에서 행성 공을 조준해 발사하고, 보드 위의 같은 등급 행성끼리 충돌시키며 더 큰 행성으로 합성한다.

이 문서는 구현 에이전트가 스스로 판단하기 어려운 게임 규칙, 아트 방향, 밸런싱 기준, UX 판단, 검증 기준을 명시한다.

## 2. 게임 컨셉

`Planet Pool Merge`는 풀 게임의 조준/반사 감각과 Suika 계열 drop merge 게임의 성장 보상을 결합한 캐주얼 물리 퍼즐이다.

핵심 장면은 첨부 레퍼런스처럼 세로 화면 중앙에 보드가 있고, 하단 고정 발사대에서 행성 공을 쏘는 구조다. 단, 포켓볼처럼 포켓에 넣는 게임이 아니라, 같은 행성끼리 충돌시켜 다음 단계로 합성하는 게임이다.

행성 단계는 다음 순서다.

1. 수성
2. 화성
3. 금성
4. 지구
5. 해왕성
6. 천왕성
7. 토성
8. 목성
9. 태양

태양은 최종 단계이며, 태양끼리 충돌해도 더 이상 합성되지 않는다.

## 3. 핵심 재미 가설

1. 조준 예측 재미: 플레이어는 발사 방향, 벽 반사, 기존 행성 배치를 보고 어떤 합성이 일어날지 예측한다.
2. 물리 우연성: 충돌, 반사, 연쇄 합성이 예상과 약간 다르게 전개되며 반복 플레이를 만든다.
3. 성장 보상: 작은 행성이 점점 큰 행성으로 바뀌고, 최종적으로 태양을 만드는 시각적 보상이 있다.
4. 공간 관리: 실패 조건은 없지만, 커지는 행성들이 보드를 채우기 때문에 좋은 각도를 찾는 압박이 생긴다.

## 4. 코어 루프

1. 3개 발사 큐를 확인한다.
2. 하단 발사대의 현재 행성을 누르고 뒤로 당겨 조준한다.
3. 손을 놓아 행성을 발사한다.
4. 행성이 벽과 다른 행성에 충돌한다.
5. 같은 등급 행성끼리 충돌하면 다음 단계 행성으로 합성된다.
6. 합성 점수와 콤보 점수가 반영된다.
7. 큐가 한 칸 당겨지고 새 랜덤 행성이 마지막 칸에 들어온다.
8. 다음 발사를 반복한다.

프로토타입에는 게임 오버가 없다. 점수, 최고 달성 행성, 합성 횟수, 태양 달성 여부로 재미를 검증한다.

## 5. 플레이 흐름

### 5.1 시작 상태

첫 플레이부터 충돌과 합성이 일어나야 하므로 빈 보드로 시작하지 않는다. 중앙에는 포켓볼의 랙처럼 초기 행성 풀이 배치된다.

기본 초기 배치:

- 수성 4개
- 화성 3개
- 금성 2개
- 지구 1개

배치 방식:

- 보드 중앙보다 약간 위쪽에 벌집형 또는 삼각 랙 형태로 배치한다.
- 행성끼리 겹치지 않도록 최소 간격을 둔다.
- 첫 발사 후 바로 충돌이 발생할 정도로 충분히 밀도 있게 배치한다.
- 해왕성 이상은 시작 배치에 넣지 않는다.

### 5.2 발사

- 발사대는 보드 하단 중앙에 고정된다.
- 플레이어는 현재 발사 행성을 누른 뒤 뒤로 당겨 조준한다.
- 손을 놓으면 드래그 방향의 반대 방향으로 행성이 발사된다.
- 드래그 거리 0~120px을 발사 파워 0~1로 정규화한다.
- 최대 발사 속도는 제한한다.
- 발사 후 짧은 쿨다운을 둔다.
- 보드 위 물리가 완전히 멈추지 않아도 다음 발사는 가능해야 한다.

### 5.3 발사 큐

- 화면에는 현재 발사 행성을 포함한 3개 큐를 보여준다.
- 발사 시 큐의 첫 번째 행성이 발사대에서 나간다.
- 발사 직후 큐가 한 칸 당겨지고, 마지막 칸에 새 랜덤 행성이 들어온다.
- 큐 보충은 Suika Game 계열의 기본 문법에 맞춰 낮은 5종 고정 후보에서 뽑는다.

랜덤 후보:

- 수성
- 화성
- 금성
- 지구
- 해왕성

확률:

- 각 20%
- 천왕성, 토성, 목성, 태양은 직접 생성되지 않고 합성으로만 등장한다.

참고: Suika Game 관련 공개 설명은 낮은 5종만 드롭 후보로 사용하는 규칙과 5종 균등 확률 설명이 일관적이다. 이 프로토타입은 그 원형에 맞춘다.

## 6. 합성 규칙

같은 단계 행성끼리 충돌하면 다음 단계 행성 하나로 합성된다.

합성 위치:

- 두 행성 중심점의 중간에 새 행성을 생성한다.

합성 속도:

- 충돌 직전 두 행성의 속도와 충돌 방향을 반영한다.
- 기본 계산은 두 속도의 평균 벡터를 사용한다.
- 평균 속도가 너무 작으면 충돌 법선 방향으로 최소 속도를 부여한다.
- 결과적으로 새 행성은 충돌 힘의 방향으로 합쳐지면서 날아가야 한다.

중복 합성 방지:

- 같은 물리 tick에서 하나의 행성이 여러 합성에 동시에 사용되지 않도록 merge lock을 둔다.
- 합성 직후 새 행성은 짧은 시간 동안 추가 합성 판정을 지연시켜 겹침으로 인한 즉시 재합성을 방지한다.
- 단, 자연스러운 연쇄 합성은 허용한다.

최종 단계:

- 태양끼리 충돌하면 일반 물리 충돌만 처리한다.

## 7. 점수와 콤보

점수는 합성과 콤보를 모두 반영한다.

기본 점수:

- 합성으로 만들어진 행성 등급이 높을수록 더 높은 점수를 준다.
- 권장 기본값: `10, 30, 70, 150, 320, 700, 1500, 3200`
- 위 점수는 화성부터 태양까지 생성될 때 순서대로 적용한다.

콤보:

- 한 번의 발사 이후 일정 시간 안에 여러 합성이 발생하면 콤보로 본다.
- 같은 발사에서 두 번째 합성부터 콤보 보너스를 적용한다.
- 권장 배율: `1.0x, 1.2x, 1.5x, 2.0x`
- 4콤보 이후는 2.0x로 제한한다.

표시:

- 상단에는 `Score`만 명확히 표시한다.
- 최고 단계 표시, 행성 진행 트랙, 실루엣 트랙은 만들지 않는다.

## 8. 화면과 UX

### 8.1 레이아웃

- 모바일 세로 화면을 우선한다.
- 기준 화면비는 9:16이다.
- 보드는 화면 중앙 대부분을 차지한다.
- 발사대는 하단 중앙에 보드와 연결된 형태로 배치한다.
- 포켓은 없다.
- 모든 보드 경계는 충돌 벽이다.

상단 UI:

- Score
- 3개 발사 큐

하단 UI:

- 발사대 행성
- 드래그 조준선
- 파워 또는 각도 표시

제외할 UI:

- 상단 행성 진행 트랙
- 미도달 단계 실루엣
- Shake 버튼
- Change Ball 버튼
- 광고 보상 버튼
- 포켓 관련 UI

### 8.2 입력

- 마우스와 터치 입력을 모두 지원한다.
- 발사대 행성을 누른 상태에서 드래그하면 조준 상태가 된다.
- 조준 중에는 반투명 조준선이 보인다.
- 조준선은 실제 발사 방향을 보여준다.
- 드래그 거리가 길수록 발사 파워가 강해진다.
- 손을 놓으면 행성이 발사된다.
- 조준 중 취소 기능은 프로토타입 필수 범위가 아니다.

## 9. 아트 방향

레퍼런스 기준은 첨부 이미지의 캐주얼 모바일 게임풍 행성 풀 보드다.

전체 톤:

- 바깥 배경은 짙은 와인색.
- 보드 프레임은 골드/나무 톤의 두꺼운 캐주얼 프레임.
- 보드 내부는 어두운 우주 배경.
- 내부 배경에는 낮은 대비의 은하 곡선 스트라이프와 작은 별 장식을 넣는다.
- 장식은 행성 식별을 방해하면 안 된다.

행성 아이콘:

- 검은색 또는 짙은 갈색의 두꺼운 외곽선.
- 플랫한 2D 원형 아이콘.
- 과한 질감 없이 2~3색 이하의 단순 패턴.
- 첨부 레퍼런스처럼 크레이터, 점무늬, 대륙, 줄무늬, 용암형 패턴을 사용한다.
- 각 행성은 크기와 패턴 모두로 구분되어야 한다.
- 단계가 올라갈수록 반지름이 명확히 커져야 한다.

권장 패턴:

- 수성: 회색, 작은 크레이터.
- 화성: 붉은색, 진한 크레이터.
- 금성: 베이지, 작은 점무늬.
- 지구: 초록 대륙과 하늘색 바다.
- 해왕성: 진한 파랑, 물결 줄무늬.
- 천왕성: 민트/청록, 분홍 또는 밝은 줄무늬.
- 토성: 노랑/황금색, 링 포함.
- 목성: 주황/갈색, 큰 줄무늬와 반점.
- 태양: 노랑/오렌지, 밝은 빛 테두리.

권장 반지름:

- 수성: 18px
- 화성: 21px
- 금성: 24px
- 지구: 28px
- 해왕성: 32px
- 천왕성: 37px
- 토성: 43px
- 목성: 50px
- 태양: 58px

보드 크기에 따라 같은 비율로 스케일링할 수 있다.

## 10. 기술 스택

권장 스택:

- Vite
- TypeScript
- PixiJS
- Matter.js

역할:

- PixiJS: 렌더링, UI, 행성 그래픽, 조준선, 배경.
- Matter.js: 원형 행성 바디, 벽 충돌, 반발, 마찰, 속도, 충돌 이벤트.
- TypeScript: 행성 데이터, 게임 상태, 점수 계산, 합성 규칙의 타입 안정성.

Three.js는 사용하지 않는다. 이 게임은 2D 물리 퍼즐이며 3D 엔진은 범위 대비 과하다.

## 11. 구현 구조

권장 모듈:

- `GameScene`: Pixi 애플리케이션, 렌더 루프, 전체 상태 전환.
- `PhysicsWorld`: Matter 엔진, 충돌 벽, 물리 step, 바디 생성/삭제.
- `PlanetFactory`: 행성 단계별 반지름, 색, 패턴, 점수, 렌더 생성.
- `Launcher`: 하단 발사대, 드래그 입력, 조준선, 발사 처리.
- `QueueSystem`: 3개 발사 큐, 낮은 5종 균등 랜덤 보충.
- `MergeSystem`: 같은 단계 충돌 감지, 합성 위치/속도 계산, merge lock.
- `ScoreSystem`: 합성 점수, 콤보 배율, 발사별 콤보 상태.
- `Hud`: Score와 3개 큐 표시.
- `BoardRenderer`: 프레임, 우주 배경, 장식, 보드 경계 시각화.

각 모듈은 한 가지 책임만 갖는다. 물리 규칙과 렌더링 규칙을 한 파일에 섞지 않는다.

## 12. 구현 태스크 분해

1. Vite TypeScript 프로젝트를 만든다.
2. PixiJS와 Matter.js를 설치한다.
3. 9:16 세로 캔버스와 반응형 스케일링을 구성한다.
4. 와인색 외부 배경, 골드/나무 프레임, 어두운 우주 보드를 그린다.
5. 포켓 없는 충돌 벽을 Matter.js에 만든다.
6. 행성 단계 데이터와 아이콘 렌더러를 만든다.
7. 중앙 초기 랙 배치를 구현한다.
8. 하단 발사대와 3개 큐를 구현한다.
9. 낮은 5종 20% 균등 랜덤 큐 보충을 구현한다.
10. 드래그 조준선과 발사 파워 계산을 구현한다.
11. Matter.js 원형 바디 발사를 구현한다.
12. 같은 단계 충돌 감지를 구현한다.
13. 합성 위치, 합성 속도, merge lock을 구현한다.
14. 합성 점수와 발사별 콤보 보너스를 구현한다.
15. Score UI를 구현한다.
16. 데스크톱과 모바일 뷰포트에서 직접 플레이 검증한다.

## 13. 에이전트 판단 가이드

아트 판단:

- 첨부 이미지의 캐주얼 행성 아이콘 스타일을 따른다.
- 실제 NASA 질감 같은 리얼한 행성 이미지는 쓰지 않는다.
- 상단 진행 트랙을 만들지 않는다.
- 행성 구분은 크기, 색, 패턴으로 해결한다.

밸런싱 판단:

- 핵심은 정확한 물리 시뮬레이션이 아니라 손맛이다.
- 공이 너무 오래 멈추지 않으면 마찰을 높인다.
- 공이 너무 빨리 죽으면 반발과 최대 발사 속도를 높인다.
- 합성 후 새 행성이 제자리에서 멈추면 최소 속도를 부여한다.
- 초반 5회 발사 안에 최소 1회 이상 합성이 자연스럽게 일어나야 한다.

UX 판단:

- 조준선은 실제 발사 방향과 일치해야 한다.
- 조준선이 너무 길어 화면을 어지럽히면 보드 경계 안에서만 보이게 한다.
- UI는 플레이 영역을 가리지 않는다.
- 발사 큐는 작아도 행성 패턴이 읽혀야 한다.
- 모바일 세로 화면에서 버튼이 없어도 발사 방법이 직관적이어야 한다.

범위 판단:

- 포켓 기능은 만들지 않는다.
- 게임 오버는 만들지 않는다.
- Shake, Change Ball, 광고 보상은 만들지 않는다.
- 행성 진행 트랙과 실루엣은 만들지 않는다.
- 저장, 랭킹, 사운드, 튜토리얼은 프로토타입 필수 범위가 아니다.

## 14. KPI

프로토타입 완성 여부는 다음 지표로 판단한다.

- 1분 안에 플레이어가 조준, 발사, 합성을 이해할 수 있다.
- 첫 5회 발사 안에 최소 1회 합성이 발생한다.
- 같은 등급 충돌 시 100% 다음 등급으로 합성된다.
- 합성 후 새 행성이 충돌 방향으로 움직인다.
- 발사 큐가 매 발사 후 정확히 한 칸씩 갱신된다.
- 점수는 합성 및 콤보에 따라 증가한다.
- 9:16 모바일 화면에서 UI 겹침이 없다.
- 데스크톱 브라우저와 모바일 크기 뷰포트에서 모두 플레이 가능하다.

## 15. 완료 체크리스트

- [ ] 첫 화면에 중앙 초기 행성 풀이 보인다.
- [ ] 하단 고정 발사대에 현재 행성이 보인다.
- [ ] 3개 발사 큐가 보인다.
- [ ] 드래그 조준선이 보인다.
- [ ] 드래그 후 놓으면 행성이 발사된다.
- [ ] 행성이 보드 벽에 튕긴다.
- [ ] 같은 단계 행성끼리 충돌하면 다음 단계로 합쳐진다.
- [ ] 합성된 행성이 충돌 힘 방향으로 움직인다.
- [ ] 태양은 최종 단계로 유지된다.
- [ ] Score가 합성 및 콤보에 따라 증가한다.
- [ ] 상단 진행 트랙이 없다.
- [ ] 실루엣 트랙이 없다.
- [ ] 포켓이 없다.
- [ ] Shake 버튼이 없다.
- [ ] Change Ball 버튼이 없다.
- [ ] 모바일 세로 화면에서 주요 UI가 겹치지 않는다.
