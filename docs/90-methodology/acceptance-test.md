# Acceptance-test Driven Spec: 완성 기준 중심 문서화

> 적용: [[../70-verification/index|70-verification]]. 출처: [[index|방법론 인덱스]].

## 목적
프로토타입이 "완성되었는지"를 객관적으로 판단. 완성 판정 기준이 없으면 에이전트는 어디서 멈출지
모른다.

## Definition of Done
| 구분 | 완료 기준 |
|---|---|
| 실행 | 로컬에서 설치 후 실행 가능 |
| 진입 | 시작 화면에서 세션 진입 가능 |
| Core Loop | 한 세션 안에서 핵심 루프 반복 가능 |
| 피드백 | 주요 행동마다 시각적 피드백 |
| 수치 표시 | 재화·점수·목표·타이머 표시 |
| 성장 | 최소 1회 성장/업그레이드/병합 경험 |
| 압박 | 장르에 맞는 실패/위험/제한/경쟁 동작 |
| 결과 | 성공/실패/세션 결과 화면 표시 |
| 재시작 | 새로고침 없이 새 세션 시작 가능 |
| 안정성 | 3분 플레이 동안 콘솔 에러 없음 |
| 검증 | KPI/디버그 패널로 핵심 지표 확인 |

## Core Loop 검증 시나리오
| 시나리오 | 검증 | 통과 기준 |
|---|---|---|
| 첫 진입 | 무엇을 할지 이해하나 | 10초 안에 첫 유효 행동 |
| 첫 보상 | 즉시 보상 있나 | 30초 안에 첫 보상 |
| 첫 성장 | 보상→성장 연결되나 | 60초 안에 첫 업그레이드/병합/해금 |
| 반복 루프 | 행동-보상-성장-새목표 반복되나 | 3분 안에 핵심 루프 5회 이상 |
| 압박 | 성장 선택 필요성 생기나 | 90초 안에 위험/제한/난이도 상승 체감 |
| 결과 | 세션 끝이 명확한가 | 실패/결과 화면 표시 |
| 재시작 | 다시 플레이 가능한가 | 결과 화면에서 재시작 |

## KPI (수치로 정의)
| KPI | 목표 | 의미 |
|---|---:|---|
| Time to First Action | ≤10초 | 조작 이해 속도 |
| Time to First Reward | ≤30초 | 보상 체감 속도 |
| Time to First Growth | ≤60초 | 성장 루프 진입 |
| Loops per Session | 3분 5회 이상 | 루프 반복성 |
| Time to First Pressure | ≤90초 | 압박 체감 |
| Restart Success | 100% | 세션 반복 가능성 |
| Console Error Count | 0 | 기본 안정성 |

## 체크리스트 작성 기준
- 주관적 표현 대신 관찰 가능한 표현 ("느껴진다"→"표시된다/가능하다/발생한다")
- 시간·횟수·조건·결과를 함께
- Core Loop 검증과 무관한 장식 기능 제외
- 완료 기준과 제외 범위를 함께 명시

## 안티패턴
- "게임이 재미있어야 한다" 같은 주관적 기준만
- "적당한 밸런스로 조정" 처럼 판단을 에이전트에 넘김
- Core Loop 완료 기준 없음
- 실행 방법은 있지만 플레이 완료 기준 없음
- 재시작·실패·결과 같은 세션 완결성 누락

## 완료 기준
- 실행·플레이·반복·결과·재시작 기준 모두 존재
- KPI가 수치로 정의됨
- 테스트 시나리오가 Core Loop와 연결됨
- 프로토타입 범위 밖 기능이 명시됨
- 체크리스트만 보고 완료 여부 판단 가능
