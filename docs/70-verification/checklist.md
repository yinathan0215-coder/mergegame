# 완료 체크리스트

> 섹션: [[70-verification/index|70 · 검증 기준]]. 자매 페이지: [[70-verification/kpi]].

## 완료 체크리스트

§15 기준. 부정 항목(없어야 하는 것)을 포함한 16개 항목을 모두 만족하면 완료로 본다.

- [ ] 첫 화면에 중앙 초기 행성 풀이 보인다.
- [ ] 하단 고정 발사대에 현재 행성이 보인다.
- [ ] 드래그 조준선이 보인다.
- [ ] 드래그 후 놓으면 행성이 발사된다.
- [ ] 행성이 보드 벽에 튕긴다.
- [ ] 같은 단계 행성끼리 충돌하면 다음 단계로 합쳐진다.
- [ ] 합성된 행성이 충돌 힘 방향으로 움직인다.
- [ ] 태양끼리는 블랙홀로 합성되고, 블랙홀은 최종 단계로 유지된다.
- [ ] Score가 충돌(벽 +1·행성 +3)·머지(등급 점수)·콤보 보너스로 증가한다.
- [ ] 머지 시 스케일 팝 + 발산 버스트 + +N 플로팅 점수 연출이 보인다.
- [ ] 상단 진행 트랙이 없다.
- [ ] 실루엣 트랙이 없다.
- [ ] 포켓이 없다.
- [ ] Shake 버튼이 없다.
- [ ] Change Ball 버튼이 없다.
- [ ] 모바일 세로 화면에서 주요 UI가 겹치지 않는다.

## 화면 구조 / HUD 체크 (2026-06-28 추가)

레이아웃·레이어·경계 개편([[../50-art-ux/layout]] · [[../50-art-ux/screen-structure]] ·
[[../30-systems/play-area-boundary]]) 검증 항목.

- [ ] 좌상단에 **나가기 버튼**(←, → Title)이 보인다(인게임 HUD엔 머니·랭킹을 두지 않는다).
- [ ] 중앙 상단에 **Score**(현재)와 **👑 최고 점수**가 보인다.
- [ ] 우상단에 **메뉴 버튼**(≡)이 보인다.
- [ ] **친구(초대)·프리미엄 버튼이 없다.**
- [ ] 보드가 **방패형**이다: 둥근 직사각형 + 하단 삼각 테이퍼 + **둥근 볼록 끝(bulge)**.
- [ ] 보드 레이어가 구분된다: 배경색(딥 네이비, 화면 전체) · 아웃라인 · 플레이그라운드 배경 이미지 · 그 사이 단색 띠.
- [ ] **발사대**(현재 행성 원형 강조 + 장식 아크)가 볼록 끝에 보인다 — 이 원/아크는 **시각 장식**.
- [ ] 발사대 주변에 **반원 충돌 영역이 없다**(충돌은 보드 outline뿐).
- [ ] **플레이 영역**(사각형)과 발사대가 **분리 선**으로 나뉜다.
- [ ] 발사된 행성은 분리 선을 **위로 통과**한다.
- [ ] 플레이 영역에 들어간 행성은 분리 선 **아래로 내려오지 않는다**(일방향 경계).
- [ ] **절대 영역:** 최대 파워 발사·강한 충돌에도 행성이 플레이 영역을 **벗어나지 않는다**(이탈 0).
- [ ] 일방향 분리 선에 복귀한 행성이 **벽처럼 튕긴다**(앞에서 멈추지 않음).

## 씬 · 메타 체크 (2026-06-28 추가)

Title 로비·해금 모달·콤보·메타 경제 검증 항목([[../20-core-loop/screen-flow]] ·
[[../50-art-ux/title-screen]] · [[../30-systems/tier-unlock]] · [[../30-systems/scoring-combo]] ·
[[../30-systems/meta-economy]]).

- [ ] **Title 로비**가 보인다: 태양계 공전 배경 + 👑최고/현재 점수 + 사이드 4버튼(일일 미션·상점·출석·행운의 돌림판) + Infinite/Stage 모드 토글.
- [ ] Play를 누르면 **Pool In-Game으로 전환**되고, HUD 나가기(←)로 Title로 돌아온다.
- [ ] 신규 등급이 처음 생성되면 **해금 소개 모달**이 뜨고 일시정지되며, OK로 닫힌다(첫 모달 = 해왕성).
- [ ] 연속 머지 시 보드 중앙에 **"Combo / N" 카운터**가 뜨고, 5단위마다 **+N 콤보 보너스**가 점수에 가산된다.
- [ ] 코인 지갑·일일 미션·출석·행운의 돌림판·상점 **팝업이 열리고 닫힌다**(미션/출석은 KST 자정 리셋·localStorage 영속).

## 평가자 관점 매핑

각 항목은 과제 요구(①~⑤)와 산출물(`game/`)을 잇는다. Phase별 완료 판정은
[[../60-implementation/task-breakdown]]과 동기화한다. 부정 항목(진행 트랙·실루엣·포켓·Shake·
Change Ball 없음)은 범위 판단의 검증선이며, 해당 범위 결정은 [[../50-art-ux/index|아트·UX]]와
[[../10-concept/index|컨셉]]에 둔다.
